# Phase 1 -- UI Design Contract

> Visual and interaction contract for Phase 1 (Foundation). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase 1 has NO user-facing screens.** This contract defines the *foundational theme system* that all downstream phases consume: colors, typography with CJK font fallback, shape tokens, spacing scale, splash screen spec, and edge-to-edge system bar handling. The only "UI" Phase 1 delivers is the splash screen and a post-splash placeholder composable.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Manual (Android Compose + MIUIX) |
| Component library | MIUIX (`top.yukonga.miuix.kmp:miuix-ui:0.9.1`) |
| Icon library | Material Icons Extended (via Compose BOM 2026.04.01 -- declared for Phase 2+) |
| Font primary | System sans-serif (MIUI default) with CJK fallback |
| Font CJK fallback | Subsetted Noto Sans SC via `Typeface.CustomFallbackBuilder` |

### Theme Architecture

```
ZhiYuTheme(themeMode: ThemeMode)       // our wrapper composable
  |
  +-- MaterialTheme (colorScheme, typography, shapes)     // Material3 layer for Scaffold/NavigationBar (Phase 2+)
  |     colorScheme: mapped from MIUIX Colors + MIUI warm orange accent (#FF6B35)
  |     typography:  14-text-style scale with CJK FontFamily applied to ALL styles
  |     shapes:      MIUI rounded corners (16dp-24dp cards, 28dp bottom sheets)
  |
  +-- MiuixTheme(colors, textStyles)   // MIUIX layer for MIUIX components
        colors:     lightColorScheme() / darkColorScheme() with primary overridden to #FF6B35
        textStyles: defaultTextStyles() with CJK-coded FontFamily set on every style
```

**Rationale for dual theme layer:** MIUIX does NOT wrap MaterialTheme (per source analysis -- it uses `CompositionLocalProvider` directly). Material3 components (`Scaffold`, `NavigationBar`, `NavigationBarItem`) from Phase 2+ require MaterialTheme context. Nesting MaterialTheme outside MiuixTheme ensures both component sets see correct theme values.

## Spacing Scale

Declared in `Dp` (Compose `dp` unit). Based on 8-point grid.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4.dp | Icon gaps, inline padding, tight spacing |
| sm | 8.dp | Compact element spacing, chip margins |
| md | 16.dp | Default element spacing, card content padding |
| lg | 24.dp | Section padding, list item margins |
| xl | 32.dp | Layout gaps, screen edge padding |
| 2xl | 48.dp | Major section breaks, modal bottom sheet margins |
| 3xl | 64.dp | Page-level spacing, top-of-screen padding |

**Exceptions for Phase 1:** None. Spacing tokens are declared as `object Spacing` in `Dimens.kt` (or inline in `Shape.kt`) for consumption by Phase 2-5. Phase 1 does NOT use spacing tokens in any UI -- they are a forward contract.

## Typography

Declared in `sp` (Compose `sp` unit). Based on MIUIX `defaultTextStyles()` 14-style scale, with CJK font applied.

**Font setup:** `Typeface.CustomFallbackBuilder` (API 29+, available at minSdk=36) builds a glyph-level fallback chain: subsetted Noto Sans SC --> system "sans-serif". **FontFamily override contract:** Every `TextStyle` in both scales MUST include `fontFamily = cjkFontFamily` (the `FontFamily` wrapping `CustomFallbackBuilder`). This ensures Chinese text renders correctly in ALL composables, whether they use MIUIX components or Material3 components.

**Implementation pattern for MIUIX TextStyles:**

```kotlin
// Type.kt
val cjkTextStyles: TextStyles
    @Composable get() {
        val cjkFamily = rememberCJKFallbackFontFamily()
        return defaultTextStyles().copy(
            main = TextStyle(fontSize = 17.sp, fontFamily = cjkFamily),
            paragraph = TextStyle(fontSize = 17.sp, lineHeight = 1.2.em, fontFamily = cjkFamily),
            body1 = TextStyle(fontSize = 16.sp, fontFamily = cjkFamily),
            // ... repeat for all 14 styles
        )
    }
```

**Accent (warm orange #FF6B35) is reserved for:**
- Active tab indicator (bottom navigation selected state)
- Primary action buttons (FAB, "save", "create" buttons)
- Interactive link text
- Clock time display digits (Phase 3)
- Countdown digits (Phase 3)
- Splash screen background
- Toggle switch active track

**Accent is NOT used for:**
- Static decorative elements
- Icons in non-active state
- Secondary action labels
- Card borders/captions
- Placeholder text

## Shape (MIUI Rounded Corners)

Declared in `dp`. Applied via `MaterialTheme.shapes` (for Material3 layer) and passed to MIUIX components that respect shape.

| Token | Value | Usage | MIUI Rationale |
|-------|-------|-------|----------------|
| extraSmall | 4.dp | Small chips, badge backgrounds | Tight inline elements |
| small | 8.dp | Small cards, input fields, dialog corners | Compact interactive surfaces |
| medium | 16.dp | Standard cards, list items, menu surfaces | Primary card corner radius |
| large | 20.dp | Elevated cards, bottom sheet top corners | Prominent surfaces |
| extraLarge | 24.dp | Modal bottom sheets, full-screen cards | Maximum rounded surfaces |
| full | 28.dp | FAB, bottom navigation bar, action chips | MIUI-style bottom bar radius |

**Material3 shape mapping:**

```kotlin
val miuiShapes = Shapes(
    extraSmall = RoundedCornerShape(4.dp),
    small = RoundedCornerShape(8.dp),
    medium = RoundedCornerShape(16.dp),
    large = RoundedCornerShape(20.dp),
    extraLarge = RoundedCornerShape(24.dp)
)
```

Note: MIUIX components use their own shape system via `LocalIndication` and `LocalOverscrollFactory` -- they do not read `MaterialTheme.shapes`. The shape tokens above are for Material3 components and custom composables. MIUIX components render MIUI-native shapes automatically.

## System Bar (Edge-to-Edge)

Android 16 mandates edge-to-edge rendering (no opt-out). Phase 1 establishes the system bar handling contract.

| Property | Value |
|----------|-------|
| Status bar | Transparent background (Android 16 default) |
| Status bar icon tint | Dark (`SYSTEM_BAR_ICON_TINTS_LIGHT = false`) in LIGHT mode |
| Status bar icon tint | Light (`SYSTEM_BAR_ICON_TINTS_LIGHT = true`) in DARK mode |
| Navigation bar | Transparent background (Android 16 default) |
| Navigation bar icon tint | Dark in LIGHT mode, Light in DARK mode |
| System bar inset handling | `WindowInsetsControllerCompat` in `MainActivity.onCreate()` |
| Edge-to-edge API | `enableEdgeToEdge()` called in `onCreate()` between `installSplashScreen()` and `super.onCreate()` |

**Execution order (critical -- PITFALL-7 mitigation):**

```kotlin
class MainActivity : ComponentActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        val splashScreen = installSplashScreen()  // 1st
        enableEdgeToEdge()                         // 2nd (MUST be after splash, before super)
        super.onCreate(savedInstanceState)          // 3rd
        // ...
    }
}
```

Window insets for content padding are handled per-screen (Phase 2+), not at the activity level. Phase 1 establishes `Modifier.systemBarsPadding()` as the standard approach.

## Splash Screen Visual Spec

| Property | Value | Source |
|----------|-------|--------|
| Splash API | AndroidX `core-splashscreen:1.2.0` | D-10 |
| Background color | `#FF6B35` (MIUI warm orange) | D-08 |
| Icon | Adaptive icon (foreground on `#FF6B35` no-background window) | D-08 |
| Text on splash | "知屿" (app name) -- rendered inside the adaptive icon foreground layer | D-08, D-12 |
| Branding image | Not used -- icon-only splash | D-08 |
| Dismiss condition | `setKeepOnScreenCondition { !isDataStoreReady }` -- dismisses AFTER DataStore emits first value | D-09 |
| White flash prevention | Solid `#FF6B35` background in splash theme; no complex drawables; correct enableEdgeToEdge ordering | PITFALL-7 |
| Splash exit animation | Crossfade (default SplashScreen API behavior) | Default |
| `postSplashScreenTheme` | Set to app theme to prevent white flash | PITFALL-7 |

### Splash Theme (`res/values/themes.xml`)

```xml
<style name="Theme.ZhiYu.Splash" parent="Theme.SplashScreen">
    <item name="windowSplashScreenBackground">@color/miui_warm_orange</item>
    <item name="windowSplashScreenAnimatedIcon">@drawable/ic_splash_adaptive</item>
    <item name="postSplashScreenTheme">@style/Theme.ZhiYu</item>
</style>
```

### Adaptive Icon Spec

The splash icon foreground should contain:
- MIUI-style rounded square icon shape (white rounded square with ~20% corner radius)
- "知屿" text centered in the icon, in MIUIX's "main" style (17sp equivalent but scaled for launcher icon)
- Transparent/empty background (the splash window background `#FF6B35` shows through)

Alternatively, use a minimal vector drawable with just the two Chinese characters "知屿" in white/semi-transparent white, centered on the `#FF6B35` splash background.
