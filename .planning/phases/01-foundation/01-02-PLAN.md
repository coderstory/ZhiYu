<objective>
Create the complete local data layer (Room database with 5 entities + FTS4, DataStore Preferences) and Koin dependency injection wiring.

**Purpose:** Establishes all data persistence that downstream phases (Info Dashboard, Knowledge Base, Profile/Settings) consume. Room schema at version 1 with auto-migration disabled (D-07). Koin provides three module slots (AppModule == singletons, RepositoryModule == business logic, ViewModelModule == presentation wiring).

**Output:** Entity classes, DAO interfaces, Room database singleton, DataStore Preferences wrapper (AppPreferences), 3 Koin modules, ZhiYuApplication class, ThemeMode model enum.

**Critical decisions per D-04/D-05/D-07:**
- D-04: All 5 entities (Article, Category, Tag, ArticleTagCrossRef, QuickNote) created upfront
- D-05: FTS4 uses unicode61 tokenizer (ICU is UNAVAILABLE on Android SQLite — will crash at runtime)
- D-07: No auto-migration; manual migration path only
- D-06: DataStore keys: theme_mode (SYSTEM/LIGHT/DARK), last_active_tab, is_first_launch
</objective>

    Create 5 data entities, 1 FTS4 virtual table entity, and 1 TypeConverters class.

    All entities in package `com.zhiyu.app.data.local.entity`:

    **1. `ArticleEntity.kt`** — Core article entity:
    - Package: com.zhiyu.app.data.local.entity
    - @Entity(tableName = "articles")
    - PrimaryKey: id (Long, autoGenerate)
    - Fields: title (String), content (String, non-null), categoryId (Long?, nullable, FK to categories.id), createdAt (Long, epoch millis), updatedAt (Long, epoch millis)
    - ForeignKey on categoryId: CategoryEntity, ON DELETE SET NULL
    - Indices: @Index("categoryId") for join performance

    **2. `CategoryEntity.kt`** — Tree category for organizing articles:
    - @Entity(tableName = "categories")
    - PrimaryKey: id (Long, autoGenerate)
    - Fields: name (String), description (String?), parentId (Long?, self-referential FK for directory tree), sortOrder (Int, default 0)
    - ForeignKey on parentId: CategoryEntity -> id, ON DELETE CASCADE
    - Indices: @Index("parentId")

    **3. `TagEntity.kt`** — Simple tag entity:
    - @Entity(tableName = "tags")
    - PrimaryKey: id (Long, autoGenerate)
    - Fields: name (String, non-null)

    **4. `ArticleTagCrossRef.kt`** — Many-to-many junction entity:
    - @Entity(primaryKeys = ["articleId", "tagId"])
    - Fields: articleId (Long), tagId (Long)
    - ForeignKeys: both to ArticleEntity(id) and TagEntity(id), ON DELETE CASCADE

    **5. `QuickNoteEntity.kt`** — Lightweight uncategorized note:
    - @Entity(tableName = "quick_notes")
    - PrimaryKey: id (Long, autoGenerate)
    - Fields: content (String, non-null), createdAt (Long, epoch millis)

    **6. `ArticleFts.kt`** — FTS4 virtual table for full-text search:
    - @Entity(tableName = "articles_fts")
    - @Fts4(contentEntity = ArticleEntity::class, tokenizer = FtsOptions.TOKENIZER_UNICODE61)
    - CRITICAL: Use `FtsOptions.TOKENIZER_UNICODE61` NOT `TOKENIZER_ICU`. ICU tokenizer is UNAVAILABLE on Android's system SQLite. Using ICU will throw `SQLiteException: unknown tokenizer: icu` at runtime.
    - Fields: rowId (Long, @ColumnInfo("rowid")), title (String), content (String)
    - Note: The FTS4 entity mirrors ArticleEntity's title and content for the virtual table.

    **7. `Converters.kt`** — Room TypeConverters:
    - Package: com.zhiyu.app.data.local.converter
    - Provide @TypeConverter for List<String> <-> String (JSON via kotlinx.serialization or manual join/split)
    - Provide @TypeConverter for any other custom types if needed

    Design notes:
    - All ID fields use `Long` not `Int` — Room best practice for autoGenerate keys
    - Dates stored as Long (epoch millis) for simplicity; use Converters if switching to LocalDateTime
    - No @Relation annotations anywhere — Phase 4 will use JOIN queries instead to avoid N+1
    - ArticleFts is listed in the database entities array alongside data entities

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| App process -> Room database | SQLite file on local filesystem, app-private storage |
| Composable -> AppPreferences Flow | Reactive preference reads from UI layer via coroutine Flow |
| Koin module -> ZhiYuApplication | DI resolution at application startup |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-FND-03 | Tampering | Room DAO @Query | mitigate | Room validates SQL at compile time. Always use bind parameters (`:paramName`), never string concatenation in @Query. |
| T-FND-04 | Information Disclosure | Room schema export | accept | `exportSchema=true` writes schema JSON to build directory. Verify via `packaging.resources.excludes` that schema JSON is NOT included in APK. Room build artifacts stay in `build/` only. |
| T-FND-05 | Denial of Service | AppPreferences first read | mitigate | All DataStore flows use `flowOn(Dispatchers.IO)` to prevent main thread blocking (PITFALL-8). Splash screen stays visible until first emission. |
| T-FND-06 | Tampering | Koin module resolution | accept | All modules are local code. No dynamic class loading or runtime dependency injection from external sources. |
</threat_model>
